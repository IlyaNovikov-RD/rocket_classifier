## Architecture

**Production package** (`rocket_classifier/`): Inference-only. No training code.

- `features.py` ÔÇö Extracts 76 physics features per trajectory from raw `(x, y, z, time_stamp)` radar pings via finite-difference kinematics. Single source of truth for feature engineering.
- `model.py` ÔÇö `RocketClassifier` class wraps a pre-trained LightGBM model. Contains `SELECTED_FEATURES` (61 features used in production after backward elimination) and `PRODUCTION_BIASES` (threshold-tuned log-probability biases). These are the single source of truth ÔÇö never duplicate them.
- `schema.py` ÔÇö Pydantic v2 validation for raw radar data (`TrajectoryPoint`, `validate_dataframe`).
- `main.py` ÔÇö Orchestrates: load data Ôćĺ validate Ôćĺ featurize Ôćĺ predict Ôćĺ write `outputs/submission.csv`.
- `app.py` ÔÇö Streamlit demo. Downloads model from GitHub Release if not local. Uses `_extract_trajectory_features` from `features.py` directly.

**Research scripts** (`research/`): Colab GPU experiments. These import `xgboost` and other libraries not in production deps ÔÇö that's intentional (cross-model comparison). Never move these into `rocket_classifier/`.

**Model artifacts** (`weights/`): Gitignored. Downloaded from the latest GitHub Release via `download_weights.py`. Uses `releases/latest/download` URLs ÔÇö never hardcode version numbers.

**Feature caches** (`cache/`): Gitignored parquet files. Regenerated from `data/` or downloaded from release.

## Key Design Decisions

- **Metric**: `min_class_recall` ÔÇö worst-class recall. Every design choice optimises for this, not accuracy.
- **61 features, not 76**: Backward elimination during Colab training dropped 15 noise features. The 61 in `SELECTED_FEATURES` are what the model was trained on. Features like `vz_mean`, `launch_angle_elev`, `launch_z` were intentionally eliminated ÔÇö they're generated by `features.py` but not used in production.
- **Threshold biases** `[0, 1.063, 2.177]`: Applied as `argmax(log(proba) + biases)` to shift decision boundaries toward minority classes (class distribution: 69%/24%/7%).
- **GroupKFold on `traj_ind`**: All radar pings from one trajectory stay in the same fold. Prevents data leakage.
- **No training in production**: Model was trained once on Colab H100. `rocket_classifier/` only does inference.

## Linting

Ruff with `line-length = 100`, target `py312`. Intentionally suppressed: `N803`/`N806` (ML convention allows uppercase `X`, `X_train`), `C408` (Plotly uses `dict()`), `B008`.

## CI

GitHub Actions on push/PR to main: ruff check + pytest. Post-release workflow (`update-interpretability.yml`) auto-regenerates SHAP assets and uploads `submission.csv` to the release.

## Workflow Rules

- **Never modify `rocket_classifier/` for experiments** ÔÇö only after an experiment proves improvement.
- **Don't push/PR until explicitly told** the task is done.
- **Cascade check on any change**: if you update one thing (e.g., model, features, biases), check whether README, app.py, docstrings, release assets, or deps also need updating.
